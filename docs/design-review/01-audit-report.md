현재 UI는 기능적으로 동작하며 기본적인 정보 구조가 잘 잡혀 있으나, 시각적 세련미와 감성 디자인이 부족하여 "MVP 프로토타입" 수준에 머물러 있음. UI-DESIGN.md에 정의된 디자인 의도 대비 구현 완성도가 약 55% 수준.

## Critical 이슈 (즉시 수정 필요)

### C1. 하드코딩된 색상값 남용
- **위치**: 거의 모든 페이지
- **현상**: `text-[#212121]`, `text-[#616161]`, `bg-[#3F51B5]`, `text-[#9E9E9E]` 등 하드코딩된 hex 값이 수십 곳에 반복됨
- **문제**: globals.css에 CSS 변수(`--foreground`, `--muted-foreground`, `--primary` 등)가 이미 정의되어 있으나 사용하지 않음. 다크모드 전환 시 색상이 전혀 반응하지 않아 **다크모드가 완전히 깨짐**
- **영향**: 모든 텍스트, 배경, 버튼 색상이 다크모드에서 고정되어 가독성 불가

### C2. 메인 레이아웃 배경색 하드코딩
- **위치**: `(main)/layout.tsx:11` - `bg-[#FAFAFA]`
- **문제**: 다크모드에서도 밝은 배경이 유지됨. `bg-background` 사용해야 함

### C3. BottomNav 다크모드 미지원
- **위치**: `bottom-nav.tsx:20` - `bg-white/95`
- **문제**: 다크모드에서 하단 네비게이션이 흰색으로 표시되어 눈부심 유발. `bg-card/95` 또는 `bg-background/95` 사용 필요

## High 이슈 (높은 임팩트)

### H1. TopBar 컴포넌트 미사용
- **위치**: `components/layout/top-bar.tsx` 존재하지만 어떤 페이지에서도 사용되지 않음
- **현상**: 각 페이지마다 제각각 자체 헤더를 구현
- **영향**: 헤더 스타일 불일관. 뒤로가기 버튼 크기/위치가 페이지마다 다름

### H2. EmptyState 컴포넌트 미사용
- **위치**: `components/common/empty-state.tsx` 존재하지만 어떤 리스트 페이지에서도 사용하지 않음
- **현상**: 연습 기록, 레슨 노트, 곡 목록이 비어있을 때 빈 화면만 표시됨
- **영향**: 신규 사용자가 앱에 진입했을 때 행동 유도(CTA) 부재

### H3. 뒤로가기 버튼 터치 타겟 미달
- **위치**: `lessons/[id]/page.tsx:45`, `pieces/[id]/page.tsx:35`, `pieces/new/page.tsx:60`, `lessons/new/page.tsx:67`
- **현상**: 실제 터치 영역이 24x24px 수준
- **기준**: 모바일 터치 타겟 최소 44x44px 필요

### H4. 연습 히스토리 FAB 위치 문제
- **위치**: `practice/page.tsx:87`
- **현상**: FAB가 `max-w-lg` 컨테이너와 무관하게 화면 중앙에 위치. 리스트 마지막 항목을 가릴 수 있음

### H5. 로딩 스피너 일관성 부족
- **현상**: 모든 페이지에 동일한 스피너 코드가 복붙되어 있음 (약 8곳)
- **문제**: 공통 로딩 컴포넌트 없음. 스켈레톤 UI 없음

### H6. 곡 상세 페이지 정보 부족
- **위치**: `pieces/[id]/page.tsx`
- **현상**: UI-DESIGN.md에 명시된 연습 통계, 구간별 메모, 관련 레슨 노트, 최근 연습 기록 모두 누락
